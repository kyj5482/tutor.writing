# tutor.writing — 아이들을 위한 매일 Writing 튜터 에이전트

미국 8학년·6학년 아이들이 **매일 읽는 책**을 바탕으로 영어 글쓰기를 연습하고,
학년 수준에 맞는 피드백을 받으며, 게임처럼 레벨을 올려가는 Claude Code 기반
튜터입니다. 모든 기록은 md 파일로 저장됩니다.

## 사용법 (How to use)

이 리포지토리에서 Claude Code를 실행한 뒤:

| 명령 | 언제 | 무엇을 하나 |
|------|------|-------------|
| `/setup` | 처음 한 번 (아이별) | 이름·학년·읽는 책을 등록하고 프로필 생성 |
| `/today` | 매일 독서 후 | 템플릿 추천 → **예시 글 먼저 보여주기** → 아이가 작성 → 즉시 피드백 → XP 지급 → md 저장 |
| `/weekly` | 주 1회 | 일주일치 글을 모아 성장 리포트 작성 (부모님용 한국어 요약 포함) |
| `/progress` | 아무 때나 | 레벨·XP·스트릭·배지 현황 보기 |

### 하루 5~15분 플로우

1. 아이가 오늘 읽은 책과 분량을 말합니다. **조금 읽은 날도 괜찮습니다** —
   분량에 맞는 템플릿(예: Quick Write 3-2-1)이 따로 있습니다.
2. 튜터가 템플릿 2개를 제안하고, 아이가 고릅니다.
3. **같은 템플릿으로 쓴 예시 글을 먼저 보여줍니다** (6학년: *Hatchet*,
   8학년: *The Giver* 기준). 그 다음 아이가 자기 책으로 씁니다.
4. 피드백: 잘한 점 2가지 🌟 + 고칠 점 딱 1가지 🔧 + 보너스 도전 1가지 🚀.
   튜터는 절대 대신 써주지 않습니다.
5. XP 지급, 스트릭 갱신, 배지 확인 → 전부 md로 커밋.

## 포트폴리오 웹 (GitHub Pages)

아이들이 쓴 글(`## My writing`)을 Epic처럼 **책 표지 그리드**로 모아 보는 정적
웹사이트가 포함되어 있습니다. md 파일을 그대로 읽어 브라우저에서 렌더링합니다.

- **책별 보기** — 글을 쓴 책마다 표지 카드로 그룹핑 (기본 화면)
- **템플릿별 보기** — Summary, Golden Line 등 템플릿별 히스토리
- **타임라인** — 월별·날짜별 히스토리
- **Tutor feedback 토글** — 헤더의 스위치로 피드백 표시/숨김 (설정은 브라우저에 저장)
- 학생이 여러 명이면 상단 탭으로 전환

### 웹 퍼블리시 설정 (1회)

1. GitHub 리포지토리 → **Settings → Pages → Source**를 **GitHub Actions**로 변경
2. main 브랜치에 푸시하면 `.github/workflows/pages.yml`이 자동으로
   `manifest.json`(일기 목록 색인)을 생성하고 배포합니다.
3. 주소: `https://kyj5482.github.io/tutor.writing/`

매일 `/today` 세션이 커밋·푸시될 때마다 포트폴리오도 자동 갱신됩니다.

### 로컬에서 미리 보기

```bash
node scripts/build-manifest.mjs   # 일기 색인 생성
python3 -m http.server 8000       # http://localhost:8000 접속
```

## 구조

```
CLAUDE.md            튜터 에이전트의 규칙 (피드백 루브릭, 세션 플로우)
templates/           10가지 글쓰기 템플릿 — 각각 3단계(Tier) 난이도
examples/grade6.md   6학년 예시: Hatchet 1~3장 기준, 전 템플릿 시연
examples/grade8.md   8학년 예시: The Giver 1~4장 기준, 전 템플릿 시연
game/rules.md        XP·레벨(100XP당 1레벨)·스트릭·배지 규칙
students/<이름>/
  profile.md         레벨, XP, 스트릭, 배지, 스킬 Tier, 세션 로그
  journal/           매일의 글 (날짜-템플릿.md) — 아이의 원문 그대로 보존
  feedback/          주간 피드백 리포트
index.html, assets/  포트폴리오 웹사이트 (md를 읽어 렌더링)
scripts/build-manifest.mjs  journal 폴더를 스캔해 manifest.json 색인 생성
.github/workflows/pages.yml GitHub Pages 자동 배포
```

## 설계 원칙

- **예시 먼저**: 처음 보는 템플릿은 어렵습니다. 잘 알려진 학년 수준 책의
  일부를 읽었다고 가정한 예시를 먼저 보여주고 시작합니다.
- **분량 유동성**: 매일 읽는 양이 달라도 됩니다. 템플릿이 읽은 분량별로
  매칭되어 있어 짧게 읽은 날도 스트릭이 끊기지 않습니다.
- **단계별 상승**: 모든 템플릿에 Tier 1→2→3이 있고, 피드백은 항상
  "다음 한 걸음"만 요구합니다. 한 스킬을 3회가량 보여주면 Tier가 올라갑니다.
- **게임화**: XP, 레벨 10단계(Scribbler→Master Author), 스트릭 보너스
  (3/7/14/30일), 배지 12종. XP는 절대 깎이지 않습니다.
- **기록**: 모든 글과 피드백은 md로 커밋되어, 한 학기 뒤 아이의 성장을
  그대로 비교해 볼 수 있습니다.
